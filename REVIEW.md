## Что проверить

### 1. Баги и логика
- Есть ли ошибки которые сломают код в рантайме
- Edge cases которые не обработаны
- Неправильная логика или условия

### 2. Безопасность
- Уязвимости (SQL injection, XSS, незащищённые эндпоинты)
- Утечки чувствительных данных
- Неправильная работа с токенами / паролями / ключами

### 3. Архитектура
- Нарушения принятой структуры проекта из .cursorrules
- Лишние зависимости между модулями
- Дублирование кода которое стоит вынести

### 4. Качество кода
- Читаемость и понятность
- Правильная типизация
- Неиспользуемые переменные / импорты / мёртвый код

### 5. Производительность
- Очевидные узкие места
- N+1 запросы к БД
- Синхронный код там где нужен async

### 6. Стиль
- Соответствие соглашениям из .cursorrules
- Единообразие именования
