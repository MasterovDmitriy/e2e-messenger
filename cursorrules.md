## ПРОЕКТ

- **Название:** E2E Messenger
- **Описание:** Веб-мессенджер для личных чатов с end-to-end шифрованием, где сервер хранит только зашифрованные данные.
- **Стек:** Backend: FastAPI, SQLAlchemy 2.0, PostgreSQL, Alembic, WebSocket, JWT, bcrypt; Frontend: React + TypeScript + Vite + Web Crypto API; Infra: Docker Compose, Nginx.
- **Деплой:** Docker + Render (web service + managed PostgreSQL), локально через Docker Compose.

## ЯЗЫК

- Общение: русский язык.
- Код, комментарии, имена переменных/функций: английский.

## ПЛАН ПЕРЕД РАБОТОЙ

- Сложные задачи (новый модуль, архитектура): покажи план 3–5 пунктов -> жди подтверждения -> пиши код.
- Простые задачи (правка, мелкая функция): пиши сразу.

## СТАНДАРТЫ КОДА

- Python 3.12+, строгая типизация (mypy-friendly), pydantic схемы на API границах.
- React + TypeScript strict mode, функциональные компоненты и хуки.
- Crypto-операции только на клиенте; приватные ключи не отправлять на сервер.
- Сервер принимает/отдает только ciphertext и публичные ключи.
- SQL миграции только через Alembic, без ручного редактирования БД.
- API контракты версионируются через `/api/v1`.
- WebSocket payload версии должны быть явно описаны в DTO.

Обязательно:

- Не добавляй зависимости без явного запроса.
- Не меняй архитектуру без явного запроса.
- Не трогай файлы вне текущего модуля.
