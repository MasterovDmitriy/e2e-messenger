#   — покажи план 3–5 пунктов → жди подтверждения → пиши код

## Шаг 2 — Разбей ТЗ на фичи

Определи все независимые фичи проекта.
Каждая фича = один законченный блок функциональности который можно реализовать отдельно.
Выведи список в порядке реализации с кратким обоснованием порядка.

## Шаг 3 — Создай TASK_PROMPT для каждой фичи

Для каждой фичи из шага 2 создай отдельный файл `TASK_[название].md` по шаблону:

```
Реализуй фичу: [название]

## Что нужно сделать
[2-3 предложения — что должно работать на выходе]

## Файлы которые нужно создать / изменить
- `path/to/file.py` — [что делает]

## Входные данные / интерфейс
[Что принимает на вход, какой API, какие параметры]

## Ожидаемый результат
[Что происходит когда фича работает]

## Зависимости
[Какие фичи должны быть готовы до этой]

## Ограничения
- Не трогай файлы вне списка выше
- Не добавляй новые зависимости

---
После завершения фичи обнови PROGRESS.md.
```
